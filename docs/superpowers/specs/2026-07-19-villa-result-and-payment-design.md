# 包棟民宿結案公告與匯款進度設計

## 目標

將原本的投票頁轉為活動結案頁，宣布大雪星空 Starry Night 當選，停止一切投票操作，並在桌面版右側提供固定名單的靜態匯款進度。

## 頁面配置

- 桌面寬度下，整個頁面改為雙欄：原本的內容完整放在左側主欄；右側為獨立的「匯款進度」卡片。
- 右側卡片於捲動時維持在可見範圍內，讓大家隨時看得到繳費進度。
- 螢幕較窄時，雙欄自動堆疊，匯款卡片放在當選公告之後、民宿資訊之前。

## 結案公告

- 頁首標題改為「投票已截止」。
- 顯示醒目的當選卡片：「當選民宿：大雪星空 Starry Night」。
- 移除姓名選擇、民宿勾選、送出、複製與清除投票按鈕，不再載入或呼叫投票 API。
- 候選民宿與比較表保留，並在大雪星空卡片加上「當選」標籤。

## 匯款進度（靜態）

- 固定 11 人：哥布林、小埋、傑哥、企鵝、白白、CH、黑熊、叫我、雪糕、yuuu、久保。
- 表格欄位只有「姓名」「金額」「狀態」。每人金額固定為 $3,000，初始狀態皆為「未匯款」。
- 表格上方摘要顯示「已匯 0／11 人」與「累計 $0」。
- 已匯款者以綠色狀態標記；未匯款者以中性或橘色標記。
- 匯款資料存放在前端的明確常數中。日後使用者提供姓名後，更新對應的靜態狀態、重新發布網站即可。

## 驗證

- 靜態檢查確認頁面含結案文字、當選民宿、11 人匯款名單、每人 $3,000 與初始未匯狀態。
- 確認不再包含可操作的投票選單、送出按鈕或 Apps Script 投票 API。
- 檢查行動版 CSS 會堆疊主欄與匯款欄。

## 範圍限制

匯款表不連動 Google Sheet，沒有登入、使用者自行勾選或匯款日期；所有狀態由網站維護者更新。

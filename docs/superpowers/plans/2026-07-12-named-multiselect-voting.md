# 記名多選投票執行計畫

> **For implementation:** 依序執行下列工作，完成每一步的驗證後再進行下一步。

## 目標

將包棟民宿投票改為固定名單的記名多選：每位成員可勾選一間以上民宿，同一名字再次送出時以最新勾選結果覆蓋舊結果。頁面需顯示各民宿票數、已投票名單與未投票名單。

固定名單：哥布林、小埋、傑哥、企鵝、白白、CH、黑熊、叫我、雪糕、yuuu、久保。

## 實作步驟

1. 先擴充靜態檢查 `work/check-villa-page.mjs`。
   - 斷言固定名單、名稱欄位、多選欄位、已投/未投名單容器，以及新的 Apps Script 欄位與回傳資料都存在。
   - 先執行測試，確認舊版會失敗，建立改動的回歸保護。

2. 改寫 `google-apps-script/Code.gs` 的資料模型與 API。
   - 使用新的 `NamedVotes` 工作表，保留現有 `Votes` 的歷史測試資料不受影響。
   - 限制姓名必須在固定名單、`villaIds` 必須是至少一個有效民宿 ID 的 JSON 陣列。
   - 每次投票追加一筆 `Timestamp / Name / Villa IDs`；讀取時只取每個名字最後一筆，計算多選票數及 `votedNames`。
   - `clear` 以空陣列覆蓋該名字的選擇。

3. 改寫 `outputs/taichung-villa-vote.html`。
   - 以姓名下拉選單和每張卡片的核取方塊取代單選按鈕。
   - 送出前驗證已選姓名與至少一間民宿；同名再送出時使用覆蓋語意。
   - 讀取並呈現彙總票數、已投票名單、未投票名單與投票進度。
   - 瀏覽器僅保存最近使用的姓名與勾選內容，資料庫的最終結果以 Apps Script 為準。

4. 更新 `google-apps-script/README.md`，清楚說明替換程式碼、填入試算表 ID、重新部署與在 `NamedVotes` 分頁查看原始記錄的方式。

5. 執行 Node 靜態測試與 HTML/關鍵字檢查。

6. 將程式碼與說明推送到公開 GitHub 儲存庫。由於 Apps Script 不可由此工作階段直接寫入，請使用者將更新的 `Code.gs` 覆蓋到 Apps Script 後重新部署；待其提供更新訊息後，對正式端點進行讀取與一筆可回復的投票/清除驗證。

## 驗證標準

- 測試確認前端與後端合約皆含姓名、多選與名單呈現。
- Apps Script 對未在名單內的姓名、空選擇與無效 ID 都拒絕寫入。
- 同名最新紀錄覆蓋舊選擇，且民宿總票數以不同姓名計算。
- 正式網頁能載入端點的新 `totals` 與 `votedNames`。

## 已知限制

這是無登入、無密碼的熟人投票。它可防止意外重複計票，但無法阻止知道他人名稱的人冒名覆蓋投票。
